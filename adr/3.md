Get will always retrieve a matching key with the latest sequence number  
Key is currently encoded USER_PROVIDED_KEY+ 54bit SEQUENCE+ 8 bit ACTION  
Updates are sorted in memtable in descending order and may duplicate USER_PROVIDED_KEY in memtable key, tie breaker will be SEQUENCE (latest sequence wins)  
Only Insert action is returned, if latest is DELETE, record will be counted as deleted and return nothing.  

Key encoding reference                                                         
n bits:User provided key |  54 bits:sequence number |   8 bits: Action  

Skiplist will be updated to take a comparator function at instantiation to allow for flexible sorting, will be used in insert and get.  
Skiplist must return the first first gte or lte value, DB will verify that user key is a match

### Black Boxing
LSM -> DB -> Memtable -> return value

### Invariants
1. Only action type Insert records will be returned if it has the latest sequence number of any key match
1. A Delete type action will never be returned
